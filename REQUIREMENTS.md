## 1. 프로젝트 개요

**KakaoTalk Blaster**는 학원 관리자 전용 일괄 메시지 전송 도구다.  
DailyReportWizard2(DRW2)와 동일한 Firebase DB를 참조하여 등록된 학생 명단을 조회하고, 관리자가 작성한 템플릿 메시지를 학생별 카카오톡 채팅방에 순차 자동 전송한다.

### 포지셔닝

| 항목 | DailyReportWizard2 | KakaoTalk Blaster |
|---|---|---|
| 대상 | 강사 (반별) | 관리자 (전체) |
| 메시지 | AI 생성 수업 리포트 | 관리자 지정 공지/성적 |
| 학생 소스 | 로컬 config.json | Firebase config/ 직접 조회 |
| DB 쓰기 | input/, session/, obs/ | 없음 (읽기 전용) |

---

## 2. 시스템 요구사항

- **OS**: Windows 10+ / macOS 12+
- **Python**: 3.9 이상
- **의존성**: `pyautogui`, `pyperclip` (Firebase는 stdlib `urllib` 사용)
- **전제**: 카카오톡 PC 앱 설치 및 로그인 상태
- **Firebase**: DRW2와 동일한 Realtime Database URL + 경로

## 3. 주요 기능

### 3.1 Firebase 학생 명단 조회
- 설정에 입력된 Firebase URL + Path 기반으로 `config/sheets/` 노드 조회
- 시트(`M` / `T`) + 반 선택 시 해당 학급 학생 목록 로드
- 연결 테스트: 설정 저장 즉시 `config/` 노드 조회로 유효성 검증
- 로드 실패 시 상태바에 오류 메시지 표시

### 3.2 학생 선택
- 반 내 전체 학생 체크박스 목록 표시
- 전체선택 / 전체해제 버튼
- 선택 인원 수 실시간 표시

### 3.3 템플릿 관리
- `templates.json`에 로컬 저장 (Firebase 비의존)
- 템플릿 추가 / 삭제
- 템플릿 본문 직접 편집 → 즉시 저장
- 템플릿 유형 2종:
  - **일반(common)**: 공통 공지. 성적 참조 없음
  - **성적 참조(score)**: Firebase `scores/` 노드 시험 데이터 참조

### 3.4 템플릿 변수 시스템
`{변수명}` 형식으로 메시지 본문에 삽입. 미정의 변수는 그대로 유지.

| 변수 | 치환값 | 유형 |
|---|---|---|
| `{이름}` | 학생 이름 | 공통 |
| `{반}` | 학급명 | 공통 |
| `{날짜}` | 오늘 날짜 (M/D 형식) | 공통 |
| `{시험명}` | 시험 유형 + 회차 | 성적 |
| `{점수}` | 해당 학생 점수 | 성적 |
| `{만점}` | 시험 만점 | 성적 |
| `{평균}` | 반 평균 (소수점 1자리) | 성적 |
| `{최고}` | 반 최고점 | 성적 |
| `{최저}` | 반 최저점 | 성적 |
| `{백분율}` | 반 내 백분율 (0~100 정수) | 성적 |

변수 삽입 버튼: UI 하단 힌트 버튼 클릭 시 커서 위치에 삽입.

### 3.5 성적 참조
- 현재 선택된 반의 `scores/{sheet}|{cls}/` 노드 조회
- 시험 목록 드롭다운 (날짜 역순 정렬): `{날짜} {유형} {회차}` 형식 표시
- 성적 참조 템플릿 전송 시 점수 미등록 학생 자동 제외 (전송 전 안내 팝업)

### 3.6 미리보기
- 현재 템플릿 + 첫 번째 선택 학생 기준으로 치환 결과 실시간 표시
- 편집 즉시 갱신

### 3.7 카카오톡 일괄 전송
- 전송 전 대상자 목록 확인 팝업 (예/아니오)
- "예" 클릭 후 3초 대기 (카카오톡 창 활성화 시간)
- 학생별 순차 전송:
  1. 채팅방명 클립보드 복사 → 카카오톡 검색(`Ctrl/Cmd+F`) → 채팅방 진입
  2. 메시지 클립보드 복사 → 붙여넣기 → 전송(Enter)
  3. 검색창 닫기(Esc) → 다음 학생
- 채팅방명 형식: `{room_prefix}{이름}` (기본값: `"오직 {이름}"`)
- 전송 중 상태바에 진행 표시 (`전송 중... (N/전체) 오직 XXX`)
- 완료 후 팝업 + 상태바 업데이트

### 3.8 설정
- Firebase URL (Realtime Database 호스트 주소)
- Firebase Path (DB 내 경로)
- 채팅방 접두사 (기본: `"오직 "`)
- 전송 딜레이 (기본: `0.5`초, 단계별 대기 시간)
- 저장 즉시 Firebase 재연결

## 5. Firebase 데이터 구조 (읽기 전용)

DRW2 Firebase DB와 동일. **KakaoTalk Blaster는 읽기만 수행, 쓰기 없음.**

```
{firebase_path}/

  config/
    sheets/
      M/
        classes/
          {cls}/
            students: [{name: "김상덕"}, ...]
      T/ ...

  scores/
    {sheet}|{cls}/          ← URL 인코딩 (예: "M%7C3MGM")
      {testKey}/            ← "{YYYY-MM-DD}|{type}|{round}"
        type:      "주간Test" | "기출모의고사" | ...
        round:     "1" | "2" | ...
        date:      "YYYY-MM-DD"
        max_score: 100
        memo:      ""
        students:  {"김상덕": 85, "이민준": 92, ...}
```

## 6. 로컬 파일 구조

```
KakaoTalkBlaster/
├── main.py              # 진입점
├── app.py               # tkinter GUI (KakaoAdminApp)
├── firebase.py          # Firebase REST 유틸 (DRW2 공유)
├── kakao_send.py        # 카카오톡 자동 전송 (pyautogui)
├── template_engine.py   # {변수} 치환 엔진
├── settings.json        # 로컬 설정 (firebase_url 등)
├── templates.json       # 사용자 템플릿 목록
├── requirements.txt     # pyautogui, pyperclip
└── REQUIREMENTS.md      # 본 문서
```

### settings.json 구조
```json
{
  "firebase_url":  "https://xxx.firebaseio.com",
  "firebase_path": "your/db/path",
  "wait_time":     0.5,
  "room_prefix":   "오직 "
}
```

### templates.json 구조
```json
[
  {
    "name": "기본 공지",
    "type": "common",
    "body": "안녕하세요, {이름} 학부모님.\n..."
  },
  {
    "name": "성적 통보",
    "type": "score",
    "body": "안녕하세요, {이름} 학부모님.\n{시험명} 결과: {점수}/{만점}..."
  }
]
```

## 7. 제약 사항

- 카카오톡 PC 앱이 **포그라운드 활성화** 상태여야 전송 가능
- 전송 중 마우스/키보드 조작 금지 (pyautogui 충돌)
- Firebase `config/` 읽기 권한 필요 (DRW2 동일 계정/규칙 사용)
- 성적 참조 전송 시 해당 시험에 점수가 등록된 학생만 전송
- `pyautogui` / `pyperclip` 미설치 시 전송 버튼 비활성
